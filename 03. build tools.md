# Build Tools & Package Management

## What is an Artifact?

An artifact is a packaged version of your application that can be deployed to an environment.
It usually involves compiling the code, packaging it (often as a compressed file), 
and storing it somewhere safe (like an artifact repository) so it can be reused or redeployed later.

---

## Artifacts by Language

Different languages produce different types of artifacts.

* **Java** → JAR or WAR files

These are created using build tools like Maven or Gradle, which:

* Download dependencies
* Compile the code
* Package everything into a single file

To run a Java application:

```bash
java -jar <artifact_name>.jar
```

---

## JavaScript: Package Managers vs Build Tools

JavaScript works a bit differently compared to Java.

### Package Managers

Tools like npm and Yarn are **package managers**, not build tools.

They are used to:

* Manage dependencies via `package.json`
* Install required libraries locally

```bash
npm install
```

---

### Running the App

```bash
npm start
```

---

### About Artifacts in JavaScript

There’s no standard artifact format like JAR/WAR in JavaScript.

You might package your app as a `.zip` or `.tar` file, but:

* It usually contains only your source code and `package.json`
* Dependencies are not included and must be installed separately

---

## Frontend Build Process

Frontend applications usually need extra processing before deployment:

* Transpiling (modern JS → browser-compatible JS)
* Bundling files together
* Minifying/compressing for performance

This is typically done using tools like Webpack.

```bash
npm run build
```

The output is a set of optimized static files ready to be deployed.

---

## Artifact Repositories

Artifacts are often stored in tools like Artifactory.

This helps with:

* Version control
* Backup
* Easy access during deployments

---

## Docker

Docker simplifies the whole process by standardizing packaging.

Instead of different artifact types, you just create a **Docker image**.

A Docker image includes:

* Application code
* Dependencies
* Runtime environment

### Benefits

* No need to install dependencies on the target machine
* Works consistently across environments

### Note

You still typically build your app first (e.g., `npm run build` or Maven build).
